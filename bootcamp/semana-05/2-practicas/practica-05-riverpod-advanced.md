# Práctica 05: Riverpod Avanzado + Tests

## Descripción

Reescribir la lógica del carrito usando `Riverpod` (StateNotifier o StateProvider) y añadir pruebas unitarias:

- Implementar un provider para el carrito.
- Tests para añadir/quitar items y verificar el estado.
- Opcional: `hooks_riverpod` para simplificar UI.

## Requisitos mínimos

- Provider que exponga lista de items y métodos para modificarla.
- Tests que cubran las funciones críticas.

## Criterios de evaluación

- Implementación Riverpod funcional (3 pts)
- Tests (2 pts)
- Documentación y comparativa con Provider/BLoC (2 pts)
